# UnyKorn Interface Doctrine

> **We do not ship complexity. We ship one elegant, trustable, scalable money-making system at a time.**

---

## The Constitution

One system across every brand.
One visual language across every device.
One intelligent shell across every context.
Minimal by default.
Voice available everywhere.
Proof visible everywhere.
AI assist available everywhere.
Self-adjusting by intent, device, and state.
Beautiful enough to feel premium.
Simple enough to feel inevitable.

---

## The Three Laws

### 1. Show Less. Mean More.

Each surface only shows:

- what matters now
- what is alive
- what needs action
- what can be trusted
- what the AI can explain

Everything else stays hidden until summoned.

### 2. One System. Many Faces.

Not separate sites. Not separate UX philosophies. Not separate design languages.

Every property is **the same living system** with:

- one shell
- one behavior model
- one AI layer
- one device intelligence layer
- one motion language
- one liquid-glass material system
- one adaptive layout engine

Each brand is only: **theme + module map + voice profile + proof surface**.

### 3. Compression Is Perfection.

Perfect means:

- fewer visible elements
- stronger hierarchy
- deeper consistency
- smoother motion
- better adaptation
- better explanation
- stronger trust
- less friction
- less code duplication

A panther, not a parade float.

---

## The 5-Question Screen Test

Every screen must answer instantly:

| # | Question | Fail condition |
|---|----------|----------------|
| 1 | **Where am I?** | No clear context or label |
| 2 | **What is live?** | No live indicators, no system pulse |
| 3 | **What can I do?** | No obvious primary action |
| 4 | **Can I trust it?** | No proof, no verification, no status |
| 5 | **Can the AI explain or do this for me?** | No agent surface, no guidance |

If a screen fails any of these, it is too noisy or too dumb.

---

## The 3-Second Rule

Every interface must answer in under 3 seconds:

- **What is this?**
- **What can I do here?**
- **What is live right now?**
- **Where do I go next?**

If it cannot, it is too dense.

## Product Discipline

### The Filter

Every system follows this gate:

- one simple problem
- one clear product
- one or two scalable offers
- one elegant user path
- everything else parked

### The 6-Field Card (mandatory for every brand)

| Field | Rule |
|-------|------|
| **Core problem** | one sentence |
| **Main offer** | exactly one |
| **Secondary offer** | optional, max one |
| **Revenue path** | one primary path |
| **Proof surface** | one trust model |
| **Deferred ideas** | everything else |

If something does not fit the card, it waits.

### The User Experience Rule

Every system should feel:

- **simple enough** for a normal human
- **premium enough** for a banker
- **beautiful enough** for a luxury brand
- **clear enough** for instant trust
- **smart enough** to scale later

The interface should say:

> Here is the one thing.
> Here is how it works.
> Here is the proof.
> Here is how you benefit.
> Here is the next action.

## The Perfect Architecture

### Layer 1: One Core Shell

The master frame for everything:

- liquid glass material rules
- bubble navigation system
- active dock
- narration rail
- avatar/guide stage
- proof/status surfaces
- adaptive layout engine

### Layer 2: One Adaptive Brain

Decides at runtime:

- module priority
- layout compression
- device mode
- narration behavior
- recommended actions
- associated systems

### Layer 3: One Design Token System

Every property uses the same:

- spacing, blur, radius, glow, contrast
- motion curves, panel depth, type scale

→ `tokens/sovereign.css`

### Layer 4: One Module Framework

Every module has the same schema:

```json
{
  "name":             "string",
  "state":            "active | dormant | locked",
  "role":             "core | intelligence | execution | capital | rwa | trust | agentic",
  "associatedChains": ["avalanche", "polygon", "solana"],
  "supportedDevices": ["desktop", "mobile", "watch", "glasses", "xr"],
  "proofTypes":       ["hash", "timestamp", "block", "attestation"],
  "narrationCopy":    "One sentence explaining what this module does.",
  "actionSet":        ["launch", "verify", "monitor"]
}
```

### Layer 5: One Brand Skin Layer

Only these change per brand:

- palette
- tone
- labels
- module set
- voice personality
- avatar style

→ `tokens/brands/*.css`

## The Intelligence Law

The system must always know:

| Signal | Source |
|--------|--------|
| **who** the user is | wallet, session, role |
| **what** they are trying to do | route, context, intent |
| **which device** they are on | viewport, UA, input mode |
| **which module** matters most | state, frequency, urgency |
| **which chain rails** are relevant | connected chain, available routes |
| **what proof** is available | on-chain, attestation, timestamp |
| **whether the AI should speak** | complexity, first visit, error state |
| **what to hide** | irrelevant modules, empty states, locked features |

The interface is not static. It is a living resolver.

## The Behavior Law

### On phone
- fewer bubbles, larger targets, tighter text
- narration-first, bottom dock

### On desktop
- fuller field, richer depth, side dock
- proof panels, multi-surface context

### On glasses
- almost no text, voice-first, strong contrast
- one task at a time, floating assistive guide

### On watch
- pulse states, proof confirmations, alerts
- one-tap actions, short AI summaries

### On XR / spatial
- true orbiting modules, layered depth
- gesture-safe spacing, anchored AI presence
- spatial proof surfaces

Every site is for all systems, not with the same layout — with the same **intelligence and design DNA**.
