# SaaS Disclosure Model

## Purpose

Show enough to build trust and win adoption without overexposing proprietary methods.

## Disclosure tiers

### Tier 1 — Public
Use for website, README, decks, and social content.
- positioning
- top-level architecture
- benefits
- selected benchmarks
- integration narrative

### Tier 2 — Sandbox / Builder
Use for approved developers and technical evaluators.
- example routes
- SDK quickstarts
- safe test credentials
- limited operational guidance

### Tier 3 — Enterprise Review
Use for serious buyers under commercial process.
- security posture
- compliance evidence model
- SLA concepts
- policy controls
- onboarding design

### Tier 4 — Customer Operations
Use for signed customers only.
- operational runbooks
- admin controls
- reporting workflows
- support procedures

### Tier 5 — Internal Restricted
Use only inside trusted internal channels.
- anti-abuse detection
- pricing engine details
- internal architecture specifics
- strategic roadmap and attack-response logic

## Recommended rule

Never expose premium operational logic just to make public marketing feel more impressive.
